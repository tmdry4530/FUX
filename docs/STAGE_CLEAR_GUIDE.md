## v1 (Legacy)

### 1. tiny_button - 작은 버튼 찾기

**패턴:** 시각적 위계 조작 (Visual Hierarchy Manipulation)

**화면 설명:**
화려한 색상의 큰 데코이 버튼들 사이에 극도로 작고 희미한 타겟 버튼이 숨겨져 있다.

**성공 조건:**
`isTarget`인 버튼을 클릭한다.

**실패 함정:**
- 데코이 버튼 클릭 시 미스 + 데코이 2개 추가 + 위치 셔플
- 데코이 레이블이 타겟과 유사 ("동의 거부", "미동의", "부분 동의")

**핵심 전략:**
| 구분 | 타겟 (정답) | 데코이 (함정) |
|------|------------|--------------|
| 크기 | 극도로 작음 (16~24px) | 크고 눈에 띔 |
| 색상 | 회색, 낮은 투명도 | 파랑/보라/녹색 (강렬) |
| 배경 | 없거나 #f5f5f5 | 화려한 배경색 |

> **핵심:** 가장 초라하고 눈에 안 띄는 버튼이 정답이다. 화려할수록 함정.

### 2. moving_target - 움직이는 과녁

**패턴:** 인터페이스 방해 (Interface Interference)

**화면 설명:**
파란 원이 화면을 튕기며 이동. 빨강/노랑/보라/초록 데코이 원도 함께 움직인다.

**성공 조건:**
유예 시간(카운트다운) 종료 후 파란 원을 탭한다.
- `colorCycle` 모드: 원의 색이 파란색(`#1b64da`)일 때만 탭해야 성공

**실패 함정:**
- 유예 시간 중 아무 곳이나 탭 → 미스
- colorCycle 모드에서 파란색이 아닐 때 탭 → 미스
- 데코이(파란색 아닌 원) 탭 → 미스
- 300ms 내 연속 탭 (탭스팸) → 미스

**핵심 전략:**

> **핵심:** 카운트다운이 끝날 때까지 절대 탭하지 않는다. 파란색만 노린다.

- colorCycle 모드: 색이 2.5초 주기로 바뀜 → 파란색 순간을 기다렸다가 한 번에
- 데코이는 빨강/노랑/보라/초록이므로 무조건 무시
- 서두르지 말고 정확하게 한 번만 탭

### 3. modal_stack - 모달 지옥

**패턴:** 강제 지속 (Forced Continuity)

**화면 설명:**
여러 겹의 모달이 쌓여있다. 각 모달에 닫기 버튼(X 또는 "닫기")이 있다.

**성공 조건:**
모든 모달을 위에서부터 순서대로 닫는다.

**실패 함정:**
| 함정 | 설명 | 대응 |
|------|------|------|
| 순서 무시 | 위가 아닌 레이어 닫기 시도 → 미스 | 반드시 최상단부터 |
| 스크롤 잠금 | 약관 끝까지 안 내리면 닫기 불가 | 먼저 끝까지 스크롤 |
| 가짜 닫기 | X와 "닫기" 중 하나만 진짜 | 둘 다 시도, 실수 시 좌우 스왑됨 |
| "다시 보지 않기" | 체크 후 닫으면 레이어 추가 | 체크박스 절대 체크 금지 |
| confirm 모달 | "설정 적용" 클릭 → 레이어 추가 | "취소" 클릭이 정답 |

> **핵심:** 최상단부터, 스크롤 완료 후, 체크박스 무시, confirm은 "취소".

## v2

### 4. roach_motel_flow - 구독 취소 미로

**패턴:** 바퀴벌레 모텔 (Roach Motel)

**화면 설명:**
구독 취소를 시도하면 할인 제안, 혜택 안내, 이중 부정 질문, 타이핑 요구 등 여러 방해 단계를 거쳐야 한다.

**성공 조건:**
모든 스텝에서 취소 진행 링크를 찾아 클릭. 마지막 requireTyping 스텝에서 `CANCEL` 입력.

**실패 함정:**
- 모든 파란 버튼 ("계속 이용하기", "혜택 유지하기", "할인 받기") → 전부 미스
- 10초 카운트다운 만료 → 자동 미스
- 이중 부정: "네, 취소를 철회합니다" → 미스 (취소를 안 하겠다는 뜻)
- 3버튼 함정: "취소 보류", "취소 예약" → 둘 다 미스

**핵심 전략:**

```
정답 = 항상 가장 작고 희미한 회색 텍스트 링크 (fontSize 11px)
함정 = 크고 파란 버튼은 전부 함정
```

- "swap" 레이아웃 주의: 간혹 색상이 반전됨 → 텍스트 내용으로 판단
- "buried_text" 레이아웃: 본문 중간에 인라인 링크로 숨겨짐
- 카운트다운 팝업: "무시하고 계속 취소" 링크 클릭
- 마지막 스텝: 대문자 `CANCEL` 정확히 입력

> **핵심:** 파란 버튼은 전부 무시. 가장 작은 회색 링크가 항상 정답.

### 5. consent_toggle_labour - 동의 토글 노동

**패턴:** 개인정보 줄트레이닝 (Privacy Zuckering)

**화면 설명:**
N개의 토글 스위치 목록. 필수 1개 + 비필수 N-1개. 비필수를 전부 OFF로 꺼야 한다.

**성공 조건:**
비필수 토글을 전부 OFF → "설정 저장" 클릭.

**실패 함정:**
| 함정 | 설명 |
|------|------|
| "모두 허용" 버튼 | 회색 보조 버튼처럼 보이지만 클릭 시 미스 |
| 추천 설정 팝업 | 절반 OFF 달성 시 등장, "적용" 누르면 전부 ON 리셋 |
| 무작위 재활성화 | 3개 이상 OFF 후 2~3초 뒤 1~2개 자동 ON |
| 불완전 저장 | 하나라도 ON인 채 저장 → 미스 |

**핵심 전략:**

> **핵심:** "모두 허용" 절대 금지. 토글 하나씩 OFF. 추천 팝업은 "건너뛰기". 저장 전 재활성화 여부 2~3초 대기 후 확인.

### 6. hidden_reject_link - 숨겨진 거절 링크

**패턴:** 수치심 확인 (Confirmshaming)

**화면 설명:**
"프리미엄 무료 체험" 프로모션. 큰 CTA 버튼과 거의 보이지 않는 거절 링크.

**성공 조건:**
진짜 거절 링크 ("괜찮습니다"/"나중에"/"건너뛰기" 중 하나) 클릭.

**실패 함정:**
- 큰 CTA 버튼 → 미스
- 가짜 거절 링크 ("무료 체험 건너뛰기", "기본 플랜으로 계속") → 미스
- 타이머 팝업의 "가입" 버튼 → 미스

**핵심 전략:**

> **핵심:** 화면 구석의 거의 투명한 텍스트를 찾는다. 그럴듯한 거절 문구("무료 체험 건너뛰기")는 가짜. 진짜는 "괜찮습니다"/"나중에"/"건너뛰기" 중 하나. 약관이 있으면 끝까지 스크롤.

### 7. disguised_cta_grid - 위장 광고 찾기

**패턴:** 위장 광고 (Disguised Ads)

**화면 설명:**
콘텐츠 카드 그리드. 진짜 콘텐츠와 위장 광고가 섞여있다.

**성공 조건:**
진짜 콘텐츠 카드(`isDisguised=false`)를 전부 탭한다.

**실패 함정:**
- 위장 광고 카드 탭 → 미스 + "이것은 위장된 광고입니다!" 피드백
- 실수마다 위장 광고 카드 추가 + 카드 순서 셔플

**핵심 전략:**

| 구분 | 진짜 콘텐츠 | 위장 광고 |
|------|------------|----------|
| "광고" 배지 | 없음 | 우상단에 있음 (극도로 희미, 8px, opacity 0.4) |
| 제목 톤 | "무료 디자인 리소스", "지금 바로 쓰는 UI 킷" | "실전 가이드", "UX 패턴 분석 도구" |

> **핵심:** 카드 우상단 모서리를 꼼꼼히 확인. 희미한 "광고" 텍스트가 있으면 함정. 역설적으로 광고처럼 들리는 제목이 진짜 콘텐츠일 수 있다.

### 8. picker_no_search - 검색 없는 선택기

**패턴:** 의도적 불편 (Deliberate Inconvenience)

**화면 설명:**
비활성화된 검색바, 기능 없는 카테고리 탭, 긴 항목 목록에서 하나를 찾아야 한다.

**성공 조건:**
목표 항목(`targetLabel`)을 찾아 클릭.

**실패 함정:**
- 다른 항목 클릭 → 미스 + 유사 항목 2개 추가 + 목록 셔플
- 검색바와 카테고리 탭은 전부 작동하지 않음

**핵심 전략:**

> **핵심:** 검색과 탭은 완전 무시. 상단의 노란 박스에 표시된 목표를 외우고, 위에서 아래로 천천히 스크롤하며 정확히 일치하는 항목 탭.

### 9. state_feedback_broken - 거짓 피드백

**패턴:** 잘못된 상태 피드백 (Broken State Feedback)

**화면 설명:**
요청서 제출 폼. 제출하면 "제출 완료!" 토스트가 뜨지만, 실제로는 완료 처리되지 않았다.

**성공 조건:**
폼 제출 후, 우하단의 희미한 "더보기" 링크 클릭 → 상태 확인 모달에서 완료.

**실패 함정:**
- "다시 제출" 버튼 (제출 후 등장) → 미스
- 이미 제출된 상태에서 "제출" 재클릭 → 미스

**핵심 전략:**

```
1. 필드 전부 채우기
2. "제출" 한 번 클릭
3. "제출 완료!" 토스트는 거짓말 → 무시
4. "다시 제출" 버튼 절대 누르지 않기
5. 우하단의 희미한 "더보기" (opacity 0.6, 11px) 클릭
6. 상태 모달에서 "요청이 접수되었습니다" 확인 → 클리어
```

> **핵심:** 성공 토스트를 믿지 마라. "더보기"가 진짜 완료 트리거.

### 10. label_ambiguity - 라벨 혼란

**패턴:** 모호한 라벨 (Ambiguous Labels)

**화면 설명:**
질문과 좌/우 두 버튼으로 구성된 다이얼로그가 연속 출제된다.

**성공 조건:**
모든 다이얼로그에서 올바른 버튼을 선택.

**실패 함정:**

| 함정 유형 | 예시 | 정답 |
|-----------|------|------|
| 색상 반전 | 빨간 "확인"이 삭제 확인 | 빨간 버튼 |
| 아이콘 역전 | 💾가 "저장 안 함"에, 🗑️가 "저장"에 | 🗑️ 버튼 (저장) |
| 이중 부정 | "취소를 철회합니다" | 의미를 천천히 해석 |
| 감정 유도 | 🏃 나가기(빨강) vs 🏠 머무르기(파랑) | 텍스트로만 판단 |

> **핵심:** 색상과 아이콘을 절대 믿지 마라. 버튼 텍스트를 소리내어 읽고 논리적으로 판단. 실수 후 좌우가 바뀔 수 있으므로 매번 다시 읽기.

## v3 - Volume Hell

> 모든 볼륨 스테이지의 공통 성공 조건: 볼륨을 목표값 ± 허용범위 안에 맞춘다.
> 공통 실패 조건: 목표를 크게 넘겨서 반대편까지 넘어가면(overshooting) 미스.

### 11. volume_hover_slider - 유령 슬라이더

**화면:** 빈 영역에 마우스를 올려야 슬라이더가 나타남. 벗어나면 사라짐.

**전략:** 영역에 마우스를 올린 채 벗어나지 않으면서 천천히 목표로 이동.

---

### 12. volume_hyper_sensitive - 과민 슬라이더

**화면:** 일반 슬라이더처럼 보이지만 감도 2.5배. 조금만 움직여도 크게 변함.

**전략:** 슬라이더 중앙 부근에서 극도로 미세하게 조작. 실수마다 감도가 1.2배씩 증가 (최대 5.0).

---

### 13. volume_tiny_hitbox - 초박 슬라이더

**화면:** 높이 3px의 극도로 얇은 트랙.

**전략:** 가로선 위에 마우스를 정밀하게 올린 후 드래그. 실수 시 히트박스 0.9배 축소.

---

### 14. volume_hidden_icon - 숨겨진 아이콘

**화면:** 🔊 아이콘만 보임.

**전략:** 아이콘을 **0.8초 이상 길게 클릭(long press)** → 슬라이더 등장 → 조작. 실수 시 아이콘 위치 이동.

---

### 15. volume_reverse_mapping - 역방향 조작

**화면:** "← 크게 | 작게 →" 안내. 방향이 반대.

**전략:** 직관의 반대로 조작. 목표 볼륨 70이면 슬라이더를 **왼쪽** 30% 위치에.

### 16. volume_random_jump - 무작위 점프

## 공통 팁

### 난이도별 최대 미스 허용 횟수

| 난이도 | 허용 미스 |
|--------|----------|
| 1 (Very Easy) | 10회 |
| 2 (Easy) | 7회 |
| 3 (Normal) | 5회 |
| 4 (Hard) | 3회 |
| 5 (Very Hard) | 2회 |

### 실수 시 벌칙 메커니즘

대부분의 스테이지에서 실수하면 난이도가 올라갑니다:
- **wrongCloseAddsLayer**: 데코이/항목/레이어 추가
- **shuffleOnMiss**: 위치/순서 재배치
- **sensitivity 증가**: 조작 감도 상승
- **opacity 감소**: 정답이 더 안 보이게 됨

### 핵심 원칙

1. **화려한 것은 함정이다** — 크고, 파랗고, 눈에 띄는 UI는 대부분 함정
2. **작고 흐린 것이 정답이다** — 회색, 작은 글씨, 낮은 투명도가 진짜 정답
3. **서두르지 마라** — 시간 제한보다 정확도가 중요. 미스 횟수 초과가 더 빨리 실패함
4. **텍스트를 읽어라** — 색상, 아이콘, 크기를 믿지 말고 텍스트 의미를 파악
5. **실수하면 더 어려워진다** — 첫 시도에 정확하게 하는 것이 최선
